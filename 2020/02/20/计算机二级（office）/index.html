计算机二级（office）

发表于 2020-02-20 更新于 2020-03-22 分类于 随笔 阅读次数：

计算机二级大家应该不陌生，相信大家也都熬夜刷过题库，本文记录了博主考二级的经历（踩得坑）还有一些学习方法。因为考完试过了好久了才想起来写文章，所以记得有点乱，主要是分享方法，文章最后有笔记。

先说点废话

某天刷微博正好看到二级报名的通知，然后心血来潮决定报名考一下。
报名付完费之后，才发现还有26天就要考试了。当晚匆匆忙忙下载了个模拟软件开始刷题。
对了我报名的是office，因为听说office最简单，但是office的过关率并不是很高，报名系统的页面我记得可以看到本地的过关率，上海的好像是百分之二十还是三十几来着，记不得了。
继续说刷题…
想着一天一套然后刷个二十套应该没问题了。
但是做了几套之后才发现，并没有想象的那么简单！！！
先说一下幻灯片，幻灯片其实并没什么难点，大同小异，每套的题目基本都差不多。

怎么刷题

主要的难点就是Word和Excel，word大家如果做过论文格式排版的话，多做几套试题，看一看讲解也就没什么大问题，但是Excel真的是坑，要背函数的。最最关键的是我看不懂题目想让我干嘛。
对了我好像记了笔记，待会儿找一下放在最后。
刚开始刷的几套，我都是懵逼的，读不懂题目。
不过做了几套也就了解做题的套路了。
我是每晚做一套，然后不会的话看一遍解析，然后再做一遍加深印象。
按顺序做了大概十套之后，然后开始随机抽题库做。抽到简单的或者做过的差不多七八十分，抽到难度大一点的就懵逼了，看不懂题目。然后我意识到光是刷题是不够的，然后开始找教程看。主要是B站，B站真的是什么教程都有，大概看了几个晚上操作题的教程，也算是总结出一些做题方法。
紧接着离考试还有几天时间，然后开始看选择题，选择题我也是刷题，题库我大概刷了几百道，但是不会的题还是基本看不懂，看文字解析也不是很懂。然后我又开始找教学视频看，主要是看知识点，不理解的就死记。

考试当天

当天我抽到了一套新题，我当时看到题目的时候傻了，看不懂题目。
我是先做幻灯片，然后word，Excel然后选择题。
幻灯片是最简单的，没什么难度，算是比较顺利。
做到word我就傻了，题目是做一个个人简历，我之前是没有做过类似的题目的。我就开始一点一点磨，做到一半的时候，我才发现题目我理解错了！！！
然后我又从头开始做，不会做的步骤，我就跳过。我记得有一个步骤需要把文字旋转一定的角度，然后还需要渐变色的艺术字，然后我找了好久都没找到怎么做，最后放弃了。
最最让我崩溃的是Excel，我真的是看不懂题目，有些步骤我不会做我就跳过，比如通过数据生成新的文档，我就直接右键新建一个，然后改成规定的名称，设置字体什么的，我当时也不知道这样能有分不，但是真的是没办法。。。。
选择题最后五分钟做完的，然后最后一分钟才交卷。

最后

算是运气比较好吧，成绩出来竟然及格了。
二十多天就过关，也可以说是创造奇迹了吧。
大家刷题的时候一定一定要找最新的题库，新增了很多考点，而且考试时候的题量真的是很大。
平时我自己刷题一套最多也就一个小时左右就做完了，但是真正考试的时候我做了两个小时，才马马虎虎做完。
这种考试还是要提前做准备，不要像我一样，临近考试才开始刷题看教程。
最后笔记给大家，对了知乎上也有很多人分享笔记，大家可以搜一下看看。

计算机二级选择题

  1. 数据流中，用标名字的箭头表示==数据流==；流程图中，用标名字的箭头表示==控制流==。

  2. 自然连接满足的两个条件：

    1. 两者有==公共域==；
    2. 通过共域的相等值（==属性==）进行连接。

  3. 线性表存储分==顺序存储==和==链式存储==，后者比前者的存储空间要大一些。现行存储链表存储空间可以是==连续==的，也可以是==不连续==的。

  4. 程序测试的目的：发现程序中的错误。

    程序调试的任务：==诊断和改正程序中的错误==。

  5. 前序遍历过程：根、左、右；

    中序遍历过程：左、根、右；

    后续遍历过程：左、右、根。

  6. 顺序存储优点：

    1. 存储密度大；
    2. 存储空间利用率高。

    缺点：插入或删除元素时不方便。

    ==链式存储，优点和缺点与上述完全相反。==

  7. 在具有2n个根节点的完全二叉树中，叶子结点为==(2n+1)/2取整==。

  8. 软件定义阶段：

    1. 问题定义；
    2. 可行性研究；
    3. 需求分析。

  9. 栈中元素

    例：占存储空间为S(1:60)初始top=61，经一系列正常入栈和退栈操作，top=1。

    ==第1个元素==：top=60+1-1=60，==第2个元素==：top=60+1-2=59……==第N个元素==：top=60+1-N=1,N=60

  10. 长度为n的顺序表中查一元素，最坏情况比较==(n+1)/2==

    有一半的机会在表中，平均情况需比较==(3+n)/4==

  11. 构成E-R图基本要素是：实体型、属性、联系。

    1. 实体型（Entity）——矩形；
    2. 属性（Attribute）——椭圆形；
    3. 联系（Relationship）——菱形。

  12. 面向对象方法中，继承是指：==类之间共享属性和操作的机制==。

  13. 算法的时间复杂度指：==执行该算法时所需的基本算数次数==。

  14. 最坏情况下：

    ​ 冒泡排序、直接插入排序、快速排序（退化冒泡）：==n(n-1)/2==。

    ​ 希尔排序：==O(n^1.5^)==。

    无论是否最坏：

    ​ 简单插入排序：==n(n-1)/2==。

    ​ 堆排序：==O(nlog2n)==

  15. 内模式也成为存储模式，是数据库物理存储结构和存取方法的描述。

  16. 需求分析阶段：需求获取、需求分析、编写需求规格说明书、需求评审。

    1. 树的结点总数 = 所有结点的度之和+1；
    2. 满二叉树：除最后一层外，每一层所有结点都有两个子节点；
    3. 完全二叉树：除最后一层，每层结点达最大值，最后一层只缺右边若干结点。==注：==具有几个结点的完全二叉树的深度为==[log2n]+1==。
    4. 二叉树基本性质：
      1. 二叉树第k层，最多有==2^k-1^(k≥1)==个结点；
      2. 深度为m的二叉树，最多有==2^m^-1==个结点；
      3. 度为0的结点（叶子结点）总比度为2的结点多1个；
      4. 有n个结点的二叉树，深度至少为==[log2n]+1==，[log2n]取整。

  18. 黑盒测试法：等价类划分法、边界值分析法、错误推测法、因果图；

    白盒测试法：逻辑覆盖测试、基本路径测试。

  19. 数据库的四个阶段：需求分析、概念设计、逻辑设计、物理设计。

  20. 栈的初始状态为top=0，在经过入栈和退栈操作后栈中元素的个数就是==top指针==指向的位置。

  21. 每个二维表代表一个==关系==。

  22. 有序表的对分查找条件是有序表为==顺序存储==。

  23. 栈 => 先进后出

    队列 => 先进先出

  24. 数据库应用系统中的核心问题是：==数据库设计==。

  25. ==外模式==在数据库设计中反省客户对数据要求的模式。

  26. 用树型结构表示实体之间联系的模型称为==层次模型==。

  27. 计算机中，信息的最小单位是==bit==。

    计算机中数据存储容量的基本单位是==字节==。

  28. 在进行数据库逻辑设计时，可将E-R图中的属性表示为关系模式的==属性==；

    将E-R图转换为关系模式时，实体和联系都可以表示为==关系==；

    在关系数据库汇总，用来表示实体间联系是==二维表==。

  29. Word文档中，将文档目录转换为纯文本格式的最优操作是：==Ctrl+shift+F9==。

  30. USB 1.1标准接口传输率是==12MB/S==，USB 2.0传输速率为==480MB/s==。

  31. 查找替换中，*代表0个或多个字符，?代表一个字符。

  32. Window 7是==单用户多任务操作系统==。

  33. 计算机运算部件一次能同时处理的二进制数据的位数称==字长==。

  34. 在关系模型中，每一个二维表称为一个==关系==。

  35. 有序表的对分查找条件时有序表为==顺序存储==。

  36. 结构化程序包括的基本控制结构是：==顺序结构、选择结构、循环结构==。

  37. Java为计算机语言。

  38. ==有序表可以用链式存储存储在不连续的存储空间内==。

  39. 在具有2n个结点的完整二叉树中，叶子结点数为==(2n+1)/2==取整。

  40. ==算法的时间复杂度与运算算法时特定的输入有关。

  41. Today()可提取当前日期；

    Now()提取当前日期和时间；

    Date()返回代码总表示日期的数字；

    Time()返回特定时间的序列数。

  42. 面向对象具有四个基本特征：==抽象、继承、封装、多态性==。

  43. ==软件定义阶段==：

    1. 需求分析；
    2. 问题定义；
    3. 可行性研究。

  44. 结构化程序强调：清晰第一，效率第二，清晰即==程序的易读性==。

  45. ==关系数据库三大组成部分==：

    1. 关系的数据操作；
    2. 关系的数据结构；
    3. 关系的完整性约束。

  46. 每经过一次数据的交换会产生新的逆序的是==快速排序==。

  47. 第二范式是在第一范式的基础上消除了==非主属性对键的部分函数依赖==；

    第三范式是第二范式的基础上消除了==非主属性对键的传递函数依赖==。

  48. ==字长==直接反映了一台计算机的计算精度。

  49. 循环列表是队列的一种==顺序==存储结构。

  50. 数据字典（DD）所定义的对象都包含于==数据流图（DFD图）==。

  51. 构成计算机软件的是：==程序数据及相关文档==。

  52. ==软件测试实施步骤==：单元测试、集成测试、确定测试、系统测试。

  53. 根据应用目的不同，汉字编码分为==外码（输入码）、交换码（国际码）、机内码、字形码、汉字地址码==。

  54. 快速定位到最后一次编辑处的快捷键==shift+F5==。

  55. 相对引用[A1]、绝对引用[$A$1]、混合引用[$A1] [A$1]。

  56. ==幻灯片浏览视图==是可在幻灯片下方显示编号的视图。

  57. PowerPoint本身不支持==.PSD==文件格式。

  58. 数据库系统三级模式是：概念模式、内模式（物理模式）、外模式（用户模式）。

  59. 软件工程三个要素：==方法、工具、过程==。

  60. 结构化程序设计原则：==自顶向下、逐步求精、模块化、限制使用GOTO语句==。

  61. 高级筛选通常需要在工作表中设置条件区域。

  62. ==幻灯片浏览试图==主要用于对所有幻灯片进行整理编排或次序调整。

  63. 在幻灯片中，快速进入下一级文本输入状态：==Ctrl+Enter==，换行：==Shift+Enter==。

  64. 算法的有穷性是指：==算法程序的运行时间是有限的==。

  65. ==and(...)函数==的参数信息不需要用[" "]。

  66. 在栈中，栈顶指针的动态变化决定栈中元素的个数。

  67. 系统结构图的最大扇入数是指：==该模块在结构图中的直接上层模块数==。

  68. ==软件设计阶段==：软件结构设计、数据设计、接口设计、过程设计。
